# My Streaming

一个自托管的局域网视频串流服务器。指向一个存有 `.mp4` 文件的文件夹，就能在同一网络内的任何设备上观看——手机、平板、其他电脑，无需拷贝任何文件。

## 快速开始

```bash
pip install -r requirements.txt
python server.py /path/to/videos
```

服务器会打印一个 URL 和一个 ASCII 二维码。扫描二维码或在同一局域网内打开该链接即可。

不指定目录时默认为当前工作目录：

```bash
python server.py
```

## 环境要求

- Python 3.8+
- FFmpeg（必须能在 `PATH` 中找到）
- NVIDIA 显卡及 CUDA 驱动（用于硬件加速转码）

## 功能特性

- 实时 HLS 转码——无需预先转换视频格式
- 断点续播——每个视频的播放进度保存在浏览器 `localStorage` 中，关掉页面再打开会自动恢复
- 字幕支持——将 `.srt` 文件放在视频旁边，服务器会自动转换为 WebVTT 格式并通过 `<track>` 标签注入播放器
- 自定义全屏按钮——将播放器旋转 90° 以适配横屏视频
- 二维码快速连接——手机扫一扫即可访问

## 工作原理

### 为什么要串流而不是直接下载？

一个 `.mp4` 文件动辄几百 MB 甚至几 GB。如果浏览器要先下载完整文件才能播放，你可能要等好几分钟才能看到画面。串流将这个视频切成短片段（每段约 10 秒），一段一段地喂给浏览器，几乎可以即时开播。

### HLS — HTTP Live Streaming

本项目使用 Apple 的 **HLS** 协议，所有现代浏览器原生支持。HLS 只有两种核心文件：

| 文件 | 作用 |
|------|------|
| `.m3u8` 播放列表 | 一个文本清单，按顺序列出每段视频的地址和时长 |
| `.ts` 片段 | 实际的视频数据块，每段约 10 秒 |

浏览器先读取 `.m3u8` 了解有哪些片段，然后按顺序逐个请求并无缝拼接播放。

```
用户点击视频
  → 浏览器请求 /stream/{id}/playlist.m3u8
  → 拿到类似这样的播放列表：
      #EXTINF:10.0,
      segment_0.ts
      #EXTINF:10.0,
      segment_1.ts
      ...
      #EXT-X-ENDLIST
  → 浏览器按顺序请求每个 .ts 并无缝播放
```

### 转码——为什么不能直接用原始 .mp4？

你电脑里的 `.mp4` 文件可能使用各种编码格式。HLS 要求视频必须是 **H.264 编码**、音频必须是 **AAC 编码**，且包装在 `.ts` 容器中。FFmpeg 将视频实时重新编码（通过 NVIDIA NVENC 硬件加速），并将 `.ts` 片段和 `.m3u8` 播放列表写入 `temp/` 目录。

服务器随后立即返回一份预生成的静态播放列表（从一开始就包含 `#EXT-X-ENDLIST`），浏览器因此会将这个流视为**点播**（VOD）而非**直播**，从而开放自由拖拽进度条。如果某个片段 FFmpeg 还没生成出来，服务器会阻塞请求最多 60 秒等待文件出现。

### 进度条拖拽与定位

当你把进度条拖到 `2:30` 时，浏览器内置的 HLS 引擎做了一个简单运算：

```
150 秒 ÷ 每段 10 秒 = 第 15 个片段
  → 请求 /stream/{id}/segment_15.ts
  → 从该位置开始解码播放
```

这段逻辑完全由浏览器内置的 HLS 支持处理——项目的 JavaScript 代码中没有任何一行与此相关。`<video>` 标签的原生 HLS 能力自动处理 seek、片段选择和缓冲。

### 字幕

扫描器（`scanner.py`）会查找与视频同名的 `.srt` 文件。找到后，服务器将其转换为 WebVTT（`.vtt`）——浏览器支持的字幕格式——并通过 `<track>` 标签注入播放器页面。浏览器会自动将字幕时间轴与当前播放时间匹配。

### 断点续播

`player.js` 每 5 秒（以及暂停/seek 时）将当前播放位置保存到 `localStorage`。再次打开同一视频时，恢复已存储的时间戳：

```
localStorage 键名：ms-pos-{video_id_hash}
```

### 单视频请求时序

```
浏览器                            HTTP Server                     FFmpeg
  │                                   │                             │
  │── GET /play/{id} ────────────────▶│                             │
  │◀── player.html ──────────────────│                             │
  │                                   │                             │
  │── GET /stream/{id}/playlist.m3u8─▶│                             │
  │                                   │── ffprobe (获取时长)         │
  │                                   │── 生成静态 .m3u8             │
  │                                   │── 启动 FFmpeg ──────────────▶│
  │◀── playlist.m3u8 (ENDLIST) ──────│                             │
  │                                   │                             │
  │── GET /stream/{id}/segment_0.ts──▶│                             │
  │◀── segment_0.ts ─────────────────│  (已生成)                    │
  │                                   │                             │
  │── GET /stream/{id}/segment_15.ts─▶│                             │
  │   (用户拖到 2:30)                  │── 轮询等待文件 ─────────────▶│
  │                                   │   FFmpeg 逐步生成...         │
  │◀── segment_15.ts ────────────────│                             │
```

## 项目结构

```
server.py              # 入口：命令行、HTTP 服务器、二维码
server/
├── app.py             # HTTP 请求处理（路由、模板渲染）
├── transcoder.py      # FFmpeg 生命周期管理、ffprobe、静态播放列表生成
├── scanner.py         # 目录扫描（发现 .mp4 和 .srt）
├── network.py         # 局域网 IP 检测、空闲端口发现
├── subtitle.py        # SRT → WebVTT 转换
├── templates/
│   ├── index.html     # 视频列表页
│   └── player.html    # 视频播放器页
└── static/
    ├── index.css
    ├── player.css
    └── player.js      # 断点续播、自定义全屏按钮
```
